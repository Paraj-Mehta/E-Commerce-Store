# Deployment Guide

This document explains how to set up and run the E-Commerce Store project in a local development environment and outlines the production deployment configuration.

---

# Prerequisites

Ensure the following software is installed before running the project.

- Node.js (Latest LTS Version Recommended)
- npm
- Git
- MongoDB Atlas Account
- Cloudinary Account

---

# Clone the Repository

```bash
git clone https://github.com/Paraj-Mehta/E-Commerce-Store.git

cd E-Commerce-Store
```

---

# Project Structure

```
E-Commerce-Store
│
├── client
└── server
```

Both applications should be installed separately.

---

# Backend Setup

Navigate to the backend directory.

```bash
cd server
```

Install dependencies.

```bash
npm install
```

---

# Environment Variables

Create a `.env` file inside the **server** directory.

```env
MONGO_URI=your_mongodb_connection_string

JWT_SECRET=your_secret_key

CLOUDINARY_NAME=your_cloudinary_name

CLOUDINARY_KEY=your_cloudinary_api_key

CLOUDINARY_SECRET=your_cloudinary_api_secret
```

---

# Running the Backend

Start the backend server.

```bash
npm start
```

The backend will start and expose the REST API.

---

# Frontend Setup

Navigate to the frontend directory.

```bash
cd client
```

Install dependencies.

```bash
npm install
```

---

# Running the Frontend

Start the React development server.

```bash
npm run dev
```

The application will be available in your browser using the local development URL generated by Vite.

---

# Database Configuration

The application uses **MongoDB Atlas** as its primary database.

The connection is established using the connection string provided through the `MONGO_URI` environment variable.

The database stores:

- Users
- Products
- Categories
- Orders
- addToCart

---

# Cloudinary Configuration

Cloudinary is used for storing product and category images.

Image upload workflow:

```
Frontend

↓

Multer

↓

Cloudinary

↓

Image URL

↓

MongoDB
```

Only the generated Cloudinary image URL is stored in the database.

---

# Authentication

The application uses JWT-based authentication.

After a successful login:

- A JWT access token is generated.
- The token is stored in Local Storage.
- Protected API requests include:

```
Authorization: Bearer <JWT_TOKEN>
```

Access tokens expire after **1 hour**.

# Common Issues

## MongoDB Connection Error

Verify that:

- `MONGO_URI` is correct.
- Your MongoDB Atlas cluster is active.
- Your IP address is allowed in MongoDB Atlas.

---

## Cloudinary Upload Failure

Verify that:

- Cloudinary credentials are correct.
- The environment variables are loaded successfully.
- Cloudinary is properly configured in the backend.

---

## Unauthorized Requests

If protected endpoints return **401 Unauthorized**:

- Verify that the user is logged in.
- Ensure the JWT token has not expired.
- Confirm the `Authorization` header is included in the request.

---

## Images Not Loading

Verify that:

- The image was uploaded successfully.
- The Cloudinary URL is stored in MongoDB.
- The stored URL is valid.
